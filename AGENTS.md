# AGENTS.md

## Общие правила работы

Ты — AI-разработчик, работающий над frontend-проектом сайта компании Нэтворкинг.

Перед выполнением любой задачи:
1. Сначала предложи краткий план действий
2. Дождись подтверждения (если задача крупная)
3. Только потом приступай к реализации

---

## Технический стек

Используй строго:

- React
- TypeScript
- Vite (или Next.js, если явно указано)
- CSS Modules или Tailwind (без перегрузки)
- React Router (если не Next.js)

---

## Приоритеты разработки

1. Frontend в приоритете (backend не реализуем сейчас)
2. Mobile-first подход
3. Чистая и масштабируемая архитектура
4. Переиспользуемые компоненты
5. Минимум зависимостей

---

## Структура проекта

Используй следующую структуру:

src/
  app/
  pages/
  components/
  sections/
  shared/
  assets/
  styles/
  data/

---

## UI/UX требования

- Один основной CTA: "Записаться на консультацию"
- Минимум визуального шума
- Чёткая иерархия
- Простые и понятные интерфейсы
- Без перегруженных анимаций

---

## Работа с компонентами

- Делай компоненты переиспользуемыми
- Разделяй layout / sections / UI элементы
- Избегай дублирования кода
- Используй props и composition

---

## Работа с задачами

Для каждой задачи:

1. Предложи план
2. Разбей на шаги
3. Реализуй по частям

Не делай сразу весь проект.

---

## Ограничения

- Не подключать лишние библиотеки
- Не делать backend без запроса
- Не усложнять архитектуру
- Не писать "заглушки ради галочки"

---

## Проверка результата

После каждой реализации:

- Код должен компилироваться
- Не должно быть ошибок
- Компоненты должны быть адаптивными
- Верстка должна быть аккуратной

---

## Цель проекта

Создать сайт, который:

- Понятен за 5 секунд
- Показывает ценность услуг
- Приводит к заявке на консультацию
